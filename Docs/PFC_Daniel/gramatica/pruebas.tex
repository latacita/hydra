Una vez generados, gracias al uso de EMFText, el editor y el analizador para nuestra gramática, procedimos a realizar su correspondiente proceso de pruebas. Para ello escribimos diferentes ficheros de restricciones, probando tanto restricciones correctas como erróneas. Por ejemplo, para el caso de la vinculación de un fichero de restricciones con un árbol de características, se probaron tanto rutas correctas como erróneas, absolutas y relativas.

La Figura~\ref{figgrampruebas} muestra la batería de pruebas utilizadas para comprobar el funcionamiento de la gramática. Al igual que en los casos anteriores, se trata de un conjunto de pruebas exhaustivas, diseñadas para abarcar todo el conjunto de sentencias que es posible construir con el lenguaje. También se analizaron las situaciones excepcionales de sentencias sintáticamente incorrectas. Para comprobar su correcto funcionamiento, se verificó tanto que el analizador del lenguaje no daba errores al procesarlo, como que el modelo generado como salida del analizador era correcto.

También se usaron las instrucciones de las pruebas del capítulo anterior, que se pueden observar en la figura \ref{figmetains} para comprobar que el modelo generado era el mismo que el creado manualmente en el capítulo anterior.
