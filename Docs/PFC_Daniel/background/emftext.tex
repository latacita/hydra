EMFText~\cite{emftext:2009} es una herramienta para diseñar sintaxis textuales para metamodelos Ecore, siguiendo un enfoque de Ingeniería de Lenguajes Dirigida por Modelos. Utilizando EMFText podemos definir la sintaxis textual de un lenguaje software utilizando una notación similar a las de las notación BNF. La Figura~\ref{fig:sle:gramaticaGrafos} muestra un ejemplo de gramática definida en EMFText para el lenguaje de grafos presentado en la Sección~\ref{sec:intr:sle}.

La sintaxis de EMFText difiere de las sintaxis BNF en que existen una serie de directivas para asociar elementos de la sintaxis textual con metaclases, de forma que se generen instancias de dichas metaclases a medida que se procesa el código de un modelo. Por ejemplo, tal y como está definida la gramática de la Figura~\ref{fig:sle:gramaticaGrafos}, a medida que vamos definiendo los arcos estamos indicando la información tanto de su peso como de sus nodos origen y destino. De este modo, EMFText genera una instanciación de la metaclase Arco que inicializa con los datos leídos y la añade a la instancia global del metamodelo.

La gran ventaja de EMFText es permite generar, a partir de la definición de una gramática, una gran cantidad de código, liberando al programador de tareas tediosas que además en muchos casos podrían resultar complicadas. Utilizando EMFText se puede generar automáticamente: (1) un editor para nuestro lenguaje, con facilidades como coloreado de la sintaxis o autocompletado; (2) un procesador para el lenguaje capaz de generar una instancia de su correspondiente metamodelo; y (3) el código necesario para empaquetar y distribuir dicho editor como un \emph{plug-in} para Eclipse. Además, todo el código generado es completamente independiente de EMFText, por lo que puede ser ejecutado en plataformas que no tengan instalado dicha herramienta; y es personalizable. Por ejemplo, se puede modificar fácilmente el postprocesador de nuestra gramática.
